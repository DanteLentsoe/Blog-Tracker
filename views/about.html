How To Use

  1. You can POST a blog by adding a new name/text to the URL /{blogname} on the blog section. Then fill in corresponding textarea fileds : issue_title, issue_text, created_by, and optional assigned_to and status_text, to store content in the database.
  2. You can ACCESS all corresponding JSON strings of a specific blog post by passing /api/issues/{blogname} to the URL.
  3. The object saved (and returned) will include all of those fields (blank for optional no input) and also include created_on(date/time), updated_on(date/time), open(boolean, true for open, false for closed), and _id.
  4. You can make updates to the blog post, but using the _id issued, within the Home page, at the update section, in order to pass new information to the database, by adding new information to any fields in the object with a value to object said object. Returned will be 'successfully updated' or 'could not update '+_id. This should always update updated_on. If no fields are sent return 'no updated field sent'.
  5. You can also DELETE /api/issues/{projectname} with a _id to completely delete an issue. If no _id is sent return '_id error', success: 'deleted '+_id, failed: 'could not delete '+_id.
  6. You can also filter the databse information by passing any field and value in the query(ie. /api/issues/{project}?open=false). The number of field values passed is not limited

URL Being Passed New Blog Post:

/api/issues/{project}

/api/issues/{project}?open=true&assigned_to=Joe
